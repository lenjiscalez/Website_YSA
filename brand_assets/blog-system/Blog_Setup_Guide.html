01

Wie das
System
funktioniert

Du schreibst Blogartikel in Notion — deiner gewohnten Umgebung. Sobald du einen Artikel als "Published" markierst, erscheint er automatisch auf der Website. Kein manuelles Exportieren, kein Code anfassen.

Notion
Artikel schreiben
→
Netlify
Function
API Proxy
→
Website
Rendert automatisch

Technisch: Die Website hat kleine Server-Funktionen (Netlify Functions), die bei jedem Seitenaufruf die neuesten Artikel von Notion holen und anzeigen. Dein Notion-API-Key bleibt dabei sicher — er ist nie im öffentlichen Code.

Was automatisch passiert

  • Neuer Artikel in Notion → erscheint auf /blog
  • Artikel-Titel → wird zur URL /blog/dein-slug
  • Artikel unveröffentlicht → verschwindet von der Website
  • Artikel editiert → Website zeigt sofort die neue Version

Was nicht automatisch passiert

Das Einmalige Setup (Notion Datenbank, Netlify Keys, Master Prompt ausführen) muss einmal pro Projekt gemacht werden. Danach: nur noch schreiben.

03

Notion:
Datenbank
einrichten

⏱
Zeitaufwand: ca. 10 Minuten Einmalig pro Projekt. Danach nur noch Artikel schreiben.

Teil A — Notion Integration erstellen

1

Zu Notion Developers gehen

Öffne notion.so/my-integrations im Browser und logge dich ein.

2

Neue Integration erstellen

Klick auf "+ New integration". Name: z.B. YSA Blog oder Kundenname Blog.

3

API Key kopieren

Nach dem Erstellen siehst du den Internal Integration Secret (beginnt mit secret_). Diesen Key sicher speichern — er wird später in Netlify eingetragen.

4

Permissions prüfen

Die Integration braucht: "Read content". Schreibrechte sind nicht nötig.

Teil B — Blog-Datenbank erstellen

5

Neue Seite in Notion erstellen

In deinem Notion Workspace eine neue Seite anlegen, z.B. "Blog — [Kundenname]".

6

Database (Full Page) einfügen

Tippe /database und wähle "Database — Full page". Eine neue Tabelle wird erstellt.

7

Properties anlegen — EXAKT diese Namen verwenden

Die Spalten müssen exakt so heißen (Groß-/Kleinschreibung beachten):

Property-Name Notion-Typ Beschreibung
Title Title (Standard) Artikelüberschrift — bereits vorhanden
Slug Text URL-Teil: z.B. mein-erster-artikel
Published Checkbox Nur bei ✅ wird Artikel angezeigt
Date Date Veröffentlichungsdatum
Summary Text Kurze Teaser-Beschreibung (1–2 Sätze)
Cover URL Direkter Bild-Link. Empfehlung: 1600 × 900 px (16:9). Dieses eine Bild erscheint auf der Artikel-Seite, auf /blog und im Homepage-Teaser — immer als 16:9-Container. Ein Upload reicht für alle drei Stellen.
8

Database ID kopieren

Die Datenbank öffnen → auf die drei Punkte (⋯) klicken → "Copy link to view". In der URL steht die ID: notion.so/{workspace}/{DATABASE_ID}?v=... — den Teil zwischen letztem / und ?v= kopieren.

9

Integration mit Datenbank verbinden

In der Datenbank: drei Punkte (⋯) → "Add connections" → deine Integration auswählen. Ohne diesen Schritt kann die API nicht auf die Daten zugreifen.

04

Netlify:
Keys
eintragen

⏱
Zeitaufwand: ca. 5 Minuten Die API-Keys werden sicher in Netlify gespeichert — nie im Code.
1

Netlify Dashboard öffnen

Gehe zu netlify.com → das richtige Projekt auswählen.

2

Environment Variables öffnen

Im Projekt: Site configuration → Environment variables → Add a variable.

3

Erste Variable eintragen

Key: NOTION_API_KEY
Value: dein Notion Integration Secret (secret_…)

4

Zweite Variable eintragen

Key: NOTION_DATABASE_ID
Value: die Database ID aus Schritt 8 oben

5

Speichern & Deploy triggern

Auf "Save" klicken. Dann Deploys → Trigger deploy → "Clear cache and deploy site".

Sicherheitshinweis Notion API Key und Database ID niemals in Code-Dateien oder Git-Commits einfügen. Immer nur über Netlify Environment Variables — so sind sie nie öffentlich sichtbar.

05

Claude Code:
Master Prompt
ausführen

⏱
Zeitaufwand: ca. 5 Minuten Claude Code baut alles automatisch — du schaust nur zu und bestätigst.
1

Projektordner in Claude Code öffnen

Claude Code starten → Projektordner der Website öffnen (dort, wo index.html liegt).

2

Neue Konversation starten

Wichtig: Neue Session starten, damit Claude Code frisch startet.

3

Master Prompt einfügen

Den vollständigen Inhalt aus brand_assets/blog-system/MASTER_PROMPT.md kopieren und in Claude Code einfügen. Nichts anpassen — der Prompt liest alles Nötige selbst.

4

Ausführen lassen

Claude Code liest die bestehende Homepage, extrahiert das Design und baut alle Dateien. Das dauert ca. 2–4 Minuten.

5

Checkliste prüfen

Am Ende zeigt Claude Code eine Checkliste. Alle 7 Punkte müssen ✅ sein.

6

GitHub pushen

Die Änderungen zu GitHub pushen — Netlify deployed automatisch. Nach ca. 1 Minute ist die Blog-Sektion live.

Was wenn etwas nicht stimmt?

Wenn das Design nicht 100% passt oder etwas fehlt: einfach Claude Code in der gleichen Session sagen was konkret angepasst werden soll. Z.B.: "Die Blog-Section auf der Homepage braucht mehr Abstand nach oben" oder "Die Cards sollen auf Mobile untereinander stehen".

06

Einen
Artikel
schreiben

Nach dem einmaligen Setup ist das Schreiben denkbar einfach. Alles passiert in Notion.

1

Notion öffnen → Blog-Datenbank öffnen

Die Datenbank, die du im Setup erstellt hast.

2

Neuen Eintrag erstellen

Auf "+ New" klicken oder einfach in der Tabelle eine neue Zeile beginnen.

3

Properties ausfüllen

Title: Artikelüberschrift
Slug: URL-Teil in Kleinbuchstaben, Bindestriche statt Leerzeichen
    → Beispiel: Titel "Mein erster Artikel" → Slug mein-erster-artikel
Date: Datum eintragen
Summary: 1–2 Sätze Teaser
Cover: Direkter Bild-Link — 1600 × 900 px (16:9) empfohlen.
    → Dasselbe Bild wird auf der Artikel-Seite, auf /blog und im Homepage-Teaser verwendet. Ein Upload reicht.

4

Artikel schreiben

Den Artikel-Inhalt direkt in der Notion-Seite schreiben — wie einen normalen Notion-Eintrag. Überschriften, Listen, Bilder, Zitate — alles wird auf der Website korrekt dargestellt.

5

Published aktivieren

Wenn fertig: Published Checkbox ✅ aktivieren. Der Artikel erscheint sofort auf der Website — kein Deploy nötig.

08

Anhang:
Master
Prompt

Den folgenden Text vollständig kopieren und in eine neue Claude Code Session einfügen. Nichts verändern.

MASTER PROMPT — vollständig kopieren brand_assets/blog-system/MASTER_PROMPT.md 
## Kontext & Ziel

Ich möchte ein vollständiges Blog-System für diese Website bauen.
Notion dient als CMS — ich schreibe Artikel in Notion, sie erscheinen
automatisch auf der Website.

Die Website ist eine Vanilla HTML/CSS/JS Site auf Netlify
(kein Framework, kein Build-Step).

---

## Schritt 1: Lies zuerst den aktuellen Stack

Öffne index.html und extrahiere:
- Alle CSS-Variablen aus :root (Farben, Fonts, Abstände)
- Die exakte Nav-Struktur (HTML)
- Die exakte Footer-Struktur (HTML)
- Alle verwendeten Animationen (fadeUp, transitions etc.)
- Den allgemeinen Seiten-Aufbau (Padding, Border-Logik,
  Typographie-Skala)

---

## Schritt 2: Baue folgende 7 Dateien

### A) blog/index.html — Blog-Übersichtsseite (URL: /blog)
Design: Identisch zur Homepage.
- fetch('/api/get-posts') → rendert Cards dynamisch
- Jede Card: Cover-Bild (opt.) als aspect-ratio:16/9 Container,
  Datum, Titel, Summary, "Weiterlesen →"
- Card-Link → /blog/{slug}
- Loading-State + Error-State im Site-Stil
- Mobile-first, responsive

### B) blog/post.html — Einzelner Artikel (URL: /blog/{slug})
Design: Identisch zur Homepage.
- Slug aus window.location.pathname lesen
- fetch('/api/get-post?slug=' + slug)
- Cover-Bild: volle Inhaltsbreite, aspect-ratio: 16/9, object-fit: cover
- Unterstützte Notion Blocks: paragraph, heading_1/2/3,
  bulleted_list_item, numbered_list_item, image, quote,
  divider, callout, code
- Zurück-Button zu /blog

### C) Homepage-Sektion in index.html
- Vor dem Footer einfügen
- 3 neueste Artikel: fetch('/api/get-posts?limit=3')
- "Alle Artikel →" Link zu /blog
- Cards: eigene CSS-Klasse .blog-preview-card, die .work-card
  überschreibt (aspect-ratio: unset !important, padding: 0 !important)
- Cover-Bild oben in der Card als aspect-ratio:16/9 Container,
  Text-Inhalt darunter mit eigenem Padding-Block
- Wenn kein Cover vorhanden: Card zeigt nur Text, kein Leerraum

### D) netlify/functions/get-posts.js
- GET /api/get-posts?limit=N
- Filter: Published = true
- Sort: Date descending
- Gibt zurück: [{id, title, slug, date, summary, cover}]
- Env: NOTION_API_KEY, NOTION_DATABASE_ID

### E) netlify/functions/get-post.js
- GET /api/get-post?slug=…
- Filter: Slug = slug UND Published = true
- Holt Blocks und wandelt sie in HTML um
- Gibt zurück: {id, title, slug, date, summary, cover, content}

### F) _redirects (erstellen/aktualisieren — OBEN einfügen)
/api/*  /.netlify/functions/:splat  200
/blog/:slug  /blog/post.html  200

### G) netlify.toml (erstellen/aktualisieren)
[functions]
  directory = "netlify/functions"
  node_bundler = "esbuild"

## Design-Regeln
- :root Variablen aus index.html unverändert übernehmen
- Kein externes CSS-Framework
- Nav und Footer pixel-identisch zu index.html
- Mobile-first, vollständig responsive

## Bilder — Ein Cover, drei Stellen
- Alle drei Image-Positionen (Artikel-Seite, /blog-Listenansicht,
  Homepage-Teaser) nutzen dasselbe cover-Feld aus Notion
- Alle drei Container: aspect-ratio: 16/9, object-fit: cover
- Empfohlene Upload-Größe: 1600 × 900 px
- Kein Bild gesetzt → kein leerer Container, Card zeigt nur Text

## Technische Anforderungen
- Keine npm-Dependencies in Netlify Functions (nur native fetch)
- Notion API Version: 2022-06-28
- Kein API-Key im Frontend-Code
